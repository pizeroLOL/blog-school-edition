术语

照顾一些可能没听说过这些东西的人。

LaTeX

一种排版排版系统，这篇文章多指Latex语句在Markdown里的应用，比如说写数学函数什么的。

类似html，只不过尖括号变为各种反斜杠，类似这样：

$\frac{x}{y}$

显示效果长这样：\frac{x}{y}

就很好玩，也很好用，不过学习成本略高，还有一些控制大小的语句啥的。

pandoc没有控制上下标的时候为直接拿这玩意写。

Markdown

一种文档语言，就是 txt 的 pro max 版，缩写为 md，在编程领域用途广泛（特指国外，国内全 tm csdn 收费，人都麻了）。

有一些很好玩的功能，详见测试页，语法见测试页的 markdown 文档。

这篇文章也是用 md 写的——当然你实际看到的时候也可能是我用 pandoc 转换后的网页。

规整，方便，即开即用，即使未经过转换也有非常高的美观度，哪怕你直接用记事本打开都非常好看，更何况上手难度并不高。

wiki.js

bilibili上一个叫程序员鱼皮的推荐过，第一次发现docker这玩意真好用。

这是一个 csr（客户端渲染，就是把文件发到你的浏览器然后拿js渲染）的wiki系统。

蓝本，评论区及其配色想法来源，以及后面可能会加一些 Material Design 的图标啥的。

我第一个类似博客的东西就是在这上面搭建的，同样是去年六一。 整活节了属于是。

这个东西头一次让我有了博客这个概念，不过这玩意latex渲染贼折磨——喵的我为什么要在初中的时候写 latex 折磨我自己——当然我只会一点点就是了，有的时候显示不出来导致写笔记的时候及其折磨。 不过好消息是写markdown还是很舒适的。

当然现在没咋用了，我希望我现在写的这个能成为一个类似 wiki.js 的 ssr（服务端渲染，就是服务端渲染好保存为静态文件发给客户端）的东西。 因为一些东西是通用的，比如说评论，LaTeX 渲染什么的。只不过后续可能得多加点东西啥的。
